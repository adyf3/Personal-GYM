# 健身容量记录网页应用 PRD

## 1. 文档信息

| 项目 | 内容 |
| --- | --- |
| 产品名称 | 健身容量记录网页应用 |
| 产品形态 | 移动端优先 Web App / PWA |
| 目标平台 | iPhone Safari，兼容桌面浏览器 |
| 主要用户 | 希望记录日常无氧训练和有氧训练容量，并观察趋势变化的个人用户 |
| 数据策略 | 本地优先，无后端 |
| 当前版本 | V1.0 MVP |

## 2. 产品背景

用户希望通过一款轻量网页应用记录每天的健身训练量。首页按无氧训练和有氧训练录入，部位与动作名称均通过下拉选项选择，用户填写或修改每个动作对应的训练数值。应用根据记录自动计算训练容量，并在趋势页按部位和时间粒度展示容量变化。

## 3. 产品目标

### 3.1 核心目标

- 首页提供部位与动作下拉选项，降低每日录入成本。
- 首页包含训练类型、部位选择、动作选择和训练参数录入。
- 部位和动作名称均为下拉选项。
- 今日训练数据可编辑。
- 往期训练数据仅可查看，不可编辑。
- 自动计算每日训练容量、部位容量和趋势图数据。
- 趋势页通过部位和时间粒度筛选后展示两个折线图。
- 第一版不提供导出、日历同步、账号、云同步功能。

### 3.2 非目标

- 不做后端服务。
- 不做用户注册登录。
- 不做数据导出。
- 不做日历同步。
- 不做社交、排行、教练端或多人协作。
- 不做 Apple Health / Google Fit 集成。
- 不支持用户自由输入动作名称和训练部位。

## 4. 页面范围

本版本仅包含两个页面：

| 页面 | 说明 |
| --- | --- |
| 今日训练页 | 通过部位和动作下拉选项输入今天的无氧训练和有氧训练量 |
| 趋势图页 | 选择部位和时间粒度后，展示无氧与有氧两个折线图 |
| 最高记录页 | 展示所有训练动作的单日最高容量记录 |

底部导航建议仅包含两个入口：

- 今日
- 最高
- 趋势

## 5. 核心概念定义

### 5.1 训练类型

训练类型分为两类：

- 无氧训练
- 有氧训练

### 5.2 训练部位

训练部位通过下拉选项选择，第一版固定提供以下选项：

- 肩
- 胸
- 背
- 核心
- 手臂
- 腿

有氧训练也需要选择部位。当前建议爬坡和跑步默认选择为“腿”，但用户可通过部位下拉框调整归属部位。

### 5.3 动作选项库

动作名称通过下拉选项选择。动作选项由系统预置，不允许用户在 V1.0 中自由输入、新增、删除或改名。

## 7. 训练容量计算

### 7.1 无氧训练容量

无氧训练使用以下公式：

```text
单动作容量 = 重量 * 次数 * 组数
```

每日无氧容量：

```text
每日无氧容量 = 当天所有无氧动作容量之和
```

部位无氧容量：

```text
部位无氧容量 = 指定时间范围内，该部位所有无氧动作容量之和
```

### 7.2 有氧训练容量

有氧训练第一版按动作类型分别计算：

```text
爬坡容量 = 坡度 * 速度 * 时长
跑步容量 = 速度 * 时长
```

每日有氧容量：

```text
每日有氧容量 = 当天所有有氧动作容量之和
```

说明：

- 有氧容量用于趋势对比，不与无氧重量容量做绝对数值比较。
- 有氧动作默认归属为腿，因此趋势页选择“腿”时可展示有氧训练容量趋势。

## 8. 功能需求

## 8.1 页面一：今日训练页

### 8.1.1 页面目标

让用户通过下拉选项快速录入今天的训练数据。页面包含两个一级标题：

```text
一级：无氧训练 / 有氧训练
记录行：部位下拉 + 动作名称下拉 + 训练参数
```

### 8.1.2 页面结构

```text
今日训练页
├── 日期与今日汇总
├── 无氧训练
│   ├── 新增无氧训练记录
│   └── 今日无氧训练记录列表
└── 有氧训练
    ├── 新增有氧训练记录
    └── 今日有氧训练记录列表
```

### 8.1.3 无氧训练记录行

每个无氧训练记录行展示两个下拉选项和三个可编辑数字字段：

| 字段 | 是否可编辑 | 说明 |
| --- | --- | --- |
| 部位 | 是，仅今日可编辑 | 下拉选项：肩、胸、背、核心、手臂、腿 |
| 动作名称 | 是，仅今日可编辑 | 下拉选项，根据所选部位过滤 |
| 重量 | 是，仅今日可编辑 | 可输入数字，可带单位展示 |
| 次数 | 是，仅今日可编辑 | 正整数 |
| 组数 | 是，仅今日可编辑 | 正整数 |
| 容量 | 否 | 自动计算 |

行展示示例：

```text
部位：肩    动作：器械上推    120 * 12 * 4    容量 5760
```

交互规则：

- 用户选择部位后，动作名称下拉框只展示该部位对应动作。
- 用户选择动作后，系统自动带入该动作的默认重量、次数、组数。
- 如果该动作存在已确认的单日最高容量记录，系统自动带入该最高记录对应的重量、次数、组数。
- 如果该动作暂无已确认最高记录，系统自动带入动作选项库中的默认重量、次数、组数。
- 用户可继续编辑重量、次数、组数。
- 如果用户切换部位，需要清空或重置动作名称选择。

### 8.1.4 有氧训练记录行

有氧训练记录行展示部位下拉、动作名称下拉和对应可编辑参数：

| 动作 | 可编辑字段 | 容量计算 |
| --- | --- | --- |
| 爬坡 | 坡度、速度、时长 | 坡度 * 速度 * 时长 |
| 跑步 | 速度、时长 | 速度 * 时长 |

通用字段：

| 字段 | 是否可编辑 | 说明 |
| --- | --- | --- |
| 部位 | 是，仅今日可编辑 | 下拉选项：肩、胸、背、核心、手臂、腿，默认腿 |
| 动作名称 | 是，仅今日可编辑 | 下拉选项：爬坡、跑步 |
| 容量 | 否 | 自动计算 |

行展示示例：

```text
部位：腿    动作：爬坡    坡度 9    速度 2.2    13 mins
部位：腿    动作：跑步    速度 5    20 mins
```

### 8.1.5 今日可编辑规则

- 当查看日期为今天时，重量、次数、组数以及有氧参数可编辑。
- 当查看日期为今天时，部位和动作名称下拉选项可编辑。
- 编辑后自动重新计算该行容量。
- 编辑后自动更新今日部位容量和趋势图数据。
- 用户可以新增、编辑、删除今天的训练记录。

### 8.1.6 往期只读规则

- 用户可以查看往期训练数据。
- 往期训练数据仅可查看，不可编辑。
- 往期页面中所有输入框和下拉选项展示为只读状态。
- 往期页面不展示保存、删除、重置等编辑操作。
- 如果某个往期动作没有记录，显示为空值或 0，但不可补录。

### 8.1.7 日期查看

首页顶部支持日期切换：

- 默认日期为今天。
- 可切换到往期日期查看历史训练。
- 仅今天日期进入编辑模式。
- 往期日期自动进入只读模式。

### 8.1.8 汇总展示

今日训练页需要展示：

- 当前日期无氧训练总容量。
- 当前日期有氧训练总容量。
- 当前日期各部位无氧容量。
- 每个动作的单动作容量。

## 8.2 页面二：趋势图页

### 8.2.1 页面目标

用户选择训练部位和时间粒度后，查看训练容量变化趋势。页面包含两个折线图。

### 8.2.2 筛选项

趋势图页顶部提供两个必选筛选项：

| 筛选项 | 可选值 |
| --- | --- |
| 部位 | 胸、肩、背、核心、手臂、腿 |
| 时间粒度 | 日、周、月 |

默认值建议：

```text
部位：胸
时间粒度：周
```

### 8.2.3 折线图

页面展示两个折线图：

| 图表 | 数据范围 | 说明 |
| --- | --- | --- |
| 无氧训练容量趋势 | 当前选择部位 + 无氧训练 | 展示该部位无氧训练容量随时间变化 |
| 有氧训练容量趋势 | 当前选择部位 + 有氧训练 | 展示该部位有氧训练容量随时间变化 |

说明：

- 当前有氧训练默认归属部位为腿。
- 当用户选择胸、肩、背、核心或手臂时，有氧训练容量趋势可能为空。
- 当用户选择腿时，有氧训练容量趋势展示爬坡和跑步聚合后的容量趋势。

### 8.2.4 图表数据规则

当时间粒度为日：

```text
每日容量 = 当天该部位、该训练类型所有记录容量之和
```

点击日维度折线图点位时：

```text
明细数据 = 当天该部位、该训练类型下所有动作的原始训练数据
```

当时间粒度为周：

```text
每周容量 = 自然周内该部位、该训练类型所有记录容量之和
```

点击周维度折线图点位时：

```text
明细数据 = 该自然周内该部位、该训练类型下，按动作聚合后的训练容量总计
```

当时间粒度为月：

```text
每月容量 = 自然月内该部位、该训练类型所有记录容量之和
```

点击月维度折线图点位时：

```text
明细数据 = 该自然月内该部位、该训练类型下，按动作聚合后的训练容量总计
```

### 8.2.5 图表展示规则

- X 轴展示日期、周或月份。
- Y 轴展示训练容量。
- 折线图点位支持点击或长按查看该点对应的动作明细。
- 当切换部位或时间粒度时，两个折线图同步刷新。
- 无数据时展示空状态。

### 8.2.6 点位明细

当用户点击任一折线图上的单个点位时，页面需要展示该点对应的动作数据明细。展示形式可以是底部抽屉、弹层或图表下方明细列表。

日维度明细：

- 展示该日期下，当前部位和当前训练类型的所有动作数据。
- 无氧动作展示：动作名称、重量、次数、组数、容量。
- 有氧动作展示：动作名称、对应参数、容量。

周维度明细：

- 展示该自然周内，当前部位和当前训练类型的所有动作训练容量总计。
- 相同动作需要合并为一行。
- 每行展示：动作名称、训练容量总计。

月维度明细：

- 展示该自然月内，当前部位和当前训练类型的所有动作训练容量总计。
- 相同动作需要合并为一行。
- 每行展示：动作名称、训练容量总计。

明细标题规则：

```text
日：2026-06-15 动作明细
周：2026-W25 动作累计
月：2026-06 动作累计
```

### 8.2.7 空状态

当所选部位和时间粒度下没有数据时，展示：

```text
暂无该部位的训练数据
```

当全部数据不足以形成趋势时，仍展示已有点位，不强制要求达到固定训练天数。

## 8.3 页面三：最高记录页

### 8.3.1 页面目标

展示所有训练动作的单日最高容量记录，帮助用户快速了解每个动作的历史最佳训练容量。

### 8.3.2 页面内容

- 展示所有无氧训练动作。
- 展示所有有氧训练动作。
- 每个动作展示动作名称、部位、训练类型、单日最高容量、最高记录日期和对应训练参数。
- 暂无已确认记录的动作展示空状态。

### 8.3.3 更新规则

- 最高记录页仅使用已确认训练数据计算。
- 用户点击“确认今日训练”后，系统重新计算所有动作的单日最高容量。
- 如果今日某个动作的单日容量超过历史最高容量，最高记录页自动更新该动作的最高记录。
- 如果今日训练数据修改后未确认，最高记录页暂不更新。

### 8.3.4 单日最高容量规则

```text
单日最高容量 = 某动作在单个日期内所有记录容量之和的历史最大值
```

说明：

- 同一天内同一动作出现多条记录时，先合并为该动作当天总容量。
- 再将该动作每天的总容量进行比较，取最高的一天作为单日最高容量记录。

## 10. 技术方案建议

### 10.1 前端技术栈

推荐：

```text
React + TypeScript + Vite + IndexedDB + Recharts
```

选择理由：

- 适合构建轻量移动端网页应用。
- Vite 启动和构建简单。
- IndexedDB 适合本地保存结构化训练数据。
- Recharts 可以快速实现移动端折线图。

### 10.2 本地存储

推荐使用 IndexedDB，封装库使用 Dexie.js。

本版本所有训练数据保存在用户本机浏览器内，不上传服务器。

### 10.3 PWA 能力

本版本建议支持：

- 添加到 iPhone 主屏幕。
- 离线打开应用。
- 本地缓存静态资源。
- 离线查看已保存训练记录。
- 离线编辑今天的训练记录。

### 11.2 记录今日有氧训练

打开应用

点击新增有氧训练记录
选择部位

进入今日训练页

查看无氧训练容量趋势折线图

## 12. MVP 范围

### 12.1 必须有

- 两个页面：今日训练页、趋势图页。
- 今日训练页包含两个一级区域：无氧训练、有氧训练。
- 今日训练页的部位为下拉选项。
- 今日训练页的动作名称为下拉选项。
- 无氧训练部位下拉包含肩、胸、背、核心、手臂、腿。
- 无氧训练动作下拉根据所选部位联动过滤。
- 有氧训练动作下拉包含爬坡、跑步。
- 今日训练数据可编辑。
- 往期训练数据仅可查看，不可编辑。
- 无氧动作支持编辑重量、次数、组数。
- 有氧动作支持编辑对应参数。
- 自动计算单动作容量、当前日期总容量、部位容量。
- 趋势图页支持选择部位。
- 趋势图页支持选择时间粒度：日、周、月。
- 趋势图页展示两个折线图：无氧训练容量趋势、有氧训练容量趋势。
- 趋势图点位支持点击查看动作明细。
- 日维度点位展示当日所有动作数据。
- 周、月维度点位展示该周期内每个动作的训练容量总计。
- 最高记录页展示所有训练动作的单日最高容量记录。
- 点击确认今日训练后，最高记录页自动更新。
- 今日新增训练动作时，默认训练参数优先使用该动作单日最高容量记录对应参数。
- 数据本地保存，刷新页面后不丢失。

### 12.2 本版本不做

- 自由输入动作名称。
- 自由输入训练部位。
- 新增或删除下拉选项库中的动作。
- 编辑往期训练数据。
- 导出功能。
- 日历同步。
- 日历视图。
- 登录注册。
- 后端服务。
- 云同步。
- 训练提醒。

## 13. 验收标准

### 13.1 今日训练页

- 页面内清晰展示“无氧训练”和“有氧训练”两个一级标题。
- 今天日期下，部位可以通过下拉框选择。
- 今天日期下，动作名称可以通过下拉框选择。
- 选择部位后，无氧动作名称下拉框按部位过滤。
- 今天日期下，无氧动作的重量、次数、组数可编辑。
- 今天日期下，有氧动作的部位、动作名称和参数可编辑。
- 往期日期下，所有训练字段和下拉框均为只读。
- 输入数值变化后，容量计算正确并同步更新汇总。
- 刷新页面后，已保存训练数据仍然存在。

### 13.2 趋势图页

- 页面顶部可以选择部位。
- 页面顶部可以选择时间粒度：日、周、月。
- 选择不同部位后，两个折线图数据同步刷新。
- 选择不同时间粒度后，两个折线图按对应规则聚合。
- 第一个折线图展示无氧训练容量趋势。
- 第二个折线图展示有氧训练容量趋势。
- 点击日维度图表点位时，展示当日该部位、该训练类型下所有动作数据。
- 点击周维度图表点位时，展示该自然周内该部位、该训练类型下每个动作的训练容量总计。
- 点击月维度图表点位时，展示该自然月内该部位、该训练类型下每个动作的训练容量总计。
- 周、月明细中相同动作需要合并展示训练容量总计。
- 当选择非腿部位且无有氧数据时，有氧趋势图展示空状态。
- 无数据时展示空状态，不报错。

### 13.3 最高记录页

- 页面可以展示所有无氧和有氧训练动作。
- 每个动作展示单日最高容量。
- 每个动作展示最高记录日期和对应训练参数。
- 暂无已确认训练数据的动作展示空状态。
- 点击确认今日训练后，如果产生新的单日最高容量，最高记录页数据同步更新。
- 今日新增某个动作时，默认训练参数自动填充为该动作单日最高容量记录对应参数。

### 13.4 本地使用

- 应用可在 iPhone Safari 正常打开。
- 应用可添加到 iPhone 主屏幕。
- 无网络状态下可以查看已缓存应用页面。
- 训练数据默认只保存在本机浏览器内。

## 15. 风险与注意事项

- 本版本无后端，清除浏览器数据可能导致训练记录丢失。
- iPhone Safari 的 PWA 离线能力受系统策略影响，首次加载仍需要网络。
- 预置下拉选项库能提升录入效率，但灵活性较低，若训练动作变化较大，需要后续支持选项库配置。
- 有氧容量和无氧容量的单位不同，适合看自身趋势，不适合直接横向比较绝对数值。

## 16. 开放问题

- 有氧训练是否需要在趋势页独立于部位筛选展示？
- 爬坡和跑步的容量公式是否需要调整为更符合你的训练习惯？
- 今日数据是否自动保存，还是需要明确点击保存？
- 往期训练是否允许复制为今日训练记录？
